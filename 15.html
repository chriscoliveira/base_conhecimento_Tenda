PDVs todos travados ou lentidão  


O que fazer: 

  

1) Verificar o consumo do Servidor. Geralmente quando há varias conexoes SMB abertas no Servidor, ele acaba consumindo CPU e RAM, chegando ao ponto de travar o Servidor. 

Para corrigir: retire o Server da rede, assim que para de receber mensagens do PDV e demais compartilhados. Abra um terminal no Server 7 e digite: 

# sudo systemctl stop smbd nmbd. 

  

Isto ira parar o servico do SAMBA. 

Com este parado, verificar se a config do arquivo smb.conf (/etc/samba/smb.conf) está parecida com o exemplo acima. Em um caso na CT08, o arquivo estava com uma linha em um bloco errado, podendo ser um possivel ofensor. Outra situação a ser observada é a versão do Kernel dos Selfs. Caso estes estejam numa versão abaixo de 5.3, esta podera causar criar varias conexões na porta 445 (SAMBA) do Servidor 7. Caso o Self esteja com esta versão, acessar a o diretorio debs na path_comum, sub-dir install-kernel_5_3_0 e executar o script: install-kernel_5_3_0.sh. Isto ira atualizar o kernel para 5.3 ou superior. 

  

  

2) Possivel PDV com logs excedendo o tamanho do storage. Consultar a seção BASE_CONHECIMENTO\PDV e verificar informações.
