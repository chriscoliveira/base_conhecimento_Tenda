Servidor nao compartilha a path_comum 


O que fazer: 

  

Verificar se o diretorio /mnt/path_comumXX (XX siginifica Unidade: 08, 09, 13, 38, 59, etc..) existe. Caso nao exista, criar com os comandos a seguir: 

mkdir -p /mnt/path_comumXX 

  

Executar os comandos de permissão: 

# chown -R root:zanthuspdv /mnt/path_comumXX 

# chmod 775 -R /mnt/path_comumXX 

  

Para testar o compartilhamento: 

smbclient -L localhost -U zanthus. Ira solicitar a senha. Usar a mesma do usuario zanthus cadastrado no AD. A saida devera ser parecida com esta: 

  

############################################## 

Enter TENDA\zanthus's password:  

  

Sharename       Type      Comment 

---------       ----      ------- 

path_comum34    Disk      Path da CT34-PINDAMONHANGABA 

IPC$            IPC       IPC Service (SRVTCCT34 server (Samba, Ubuntu)) 

SMB1 disabled -- no workgroup available 

############################################## 

  

Caso coloque a senha ele dê erro, temos algumas situações a seguir: 

1) Resetar a senha do usuario zanthus no Server 7: smbpasswd -a zanthus. Quando solicitar a senha, usar a mesma do usuario zanthus cadastrado no AD. 

2) Revisitar o arquivo smb.conf e verificar se as informações do compartilhamento esta corretas (nome, local, usuarios permitidos, etc) 

3) Em alguns casos, precisa ser verifica se os usuarios de acesso a path_comum esta cadastrados localmente no Servidor. Para tal, acessar o arquivo /etc/passwd e procurar pelos usuarios: zanthus, admzanthus, usradmXX. Caso nao exista algum destes, cadastrar com o comando a seguir: 

# adduser usuario 

Ele ira pedir a senha e algumas informaçoes do usuario em cadastramento. Na parte de informações, caso nao queira cadastrar nada, so pressionar ENTER ate terminar. 

  

  

  

Aqui vou colocar um comparativo de como ele devera ser configurado o arquivo smb.conf (/etc/samba)
